═══════════════════════════════════════════════════════════════
  🎯 UNDERSTANDING THE DEPLOYMENT
═══════════════════════════════════════════════════════════════

How Render Deploys Your App:
────────────────────────────

1. GitHub Push Detected
   → Render receives webhook
   → Starts build process

2. Build Phase
   → Changes directory to "Root Directory" (api/)
   → Runs "Build Command" (pip install -r requirements.txt)
   → Installs dependencies

3. Start Phase
   → Runs "Start Command" (gunicorn app:app)
   → Gunicorn starts Flask app
   → Binds to port (auto-set by Render)

4. Proxy Setup
   → Render's NGINX proxy routes traffic
   → Your custom domain points to proxy
   → Proxy forwards to Gunicorn

Request Flow:
─────────────
Browser
   ↓
https://your-app.onrender.com/crops
   ↓
Render NGINX Proxy
   ↓
Gunicorn (port $PORT)
   ↓
Flask App (app.py)
   ↓
Route: @app.route('/crops')
   ↓
JSON Response

If 404 at ANY step:
→ Request reaches proxy: ✅ Render working
→ Proxy reaches Gunicorn: Check logs for "Listening at"
→ Gunicorn reaches Flask: Check for Python errors
→ Flask finds route: Check route registration
